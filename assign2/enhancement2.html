Welcome to the enhancement page for php

There are two enhancements done in the php part

Enhancement 1:

The first enhancement which was done was normalising the tables.
Normalisation, in database management, is a way to ensure less redundancy and a more reliable and accurate data structures leading to better data integrity.
When building the Attempts table, we observed that the student ID, first name and last name were repeated for each attempt that the student made. Instead of keeping this redundancy in our database, we split the Attempts table into two - Student and Attempts.

If the Student table does not already exist in our database, we create it. If it does, we insert verified student attempt records into this database only if it is the student's first attempt at the quiz. If the student record already exists, the required data are inserted into the Attempts table only.

Enhancement 2:

To limit the access into manage.php page, we have created a login.php that opens first. The access to manage.php is only allowed if correct credentials are entered into the login page.
The page also has an option to sign up where a new supervisor can register himself and his details will be stored in the database in a table named 'users'. The login page takes the username finds the corresponding passwords from the database and compares the input password with the corresponding password. If the two match, access is granted and if not, an error message is displayed.
